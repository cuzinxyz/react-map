Key Importance

Keys help React identify which items have changed, are added, or removed. Always use unique, stable keys for list items.

✅ Good: key={user.id} (unique, stable)
⚠️ Okay: key={index} (only for static lists)
❌ Bad: key={Math.random()} (not stable)

Error Boundary Best Practices

✅ Use Error Boundaries to catch JavaScript errors in component tree

✅ Display fallback UI instead of crashing the entire app

✅ Log errors for debugging and monitoring

❌ Error Boundaries do NOT catch:

• Errors in event handlers

• Errors in async code (setTimeout, promises)

• Errors during server-side rendering

• Errors thrown in the Error Boundary itself

HOC Best Practices

✅ Good use cases:

• Cross-cutting concerns (auth, logging, theming)

• Code reuse across multiple components

• Conditional rendering logic

• Data fetching patterns

❌ Avoid HOCs for:

• Simple prop manipulation (use regular functions)

• One-time use cases (create custom hooks instead)

• Complex prop transformations (prefer render props)

⚠️ Modern alternatives:

• Custom hooks (preferred for stateful logic)

• Render props (for flexible component composition)

• Context API (for global state)

Suspense & Lazy Loading Best Practices

✅ Good use cases:

• Large components that aren't immediately needed

• Route-based code splitting

• Heavy third-party libraries (charts, editors)

• Components behind user interactions

✅ Loading state best practices:

• Use skeleton screens for better UX

• Match loading state size to actual content

• Provide meaningful loading messages

• Handle loading errors gracefully

⚠️ Considerations:

• Don't over-split - balance bundle size vs requests

• Consider preloading for critical components

• Test loading states thoroughly

• Handle network failures

🔄 Modern patterns:

• Nested Suspense boundaries

• Concurrent features with Suspense

• Data fetching with Suspense (React 18+)

Advanced Hooks Summary

useLayoutEffect:

• Runs synchronously after DOM mutations

• Use for DOM measurements, preventing flicker

• Blocks browser paint until complete

useImperativeHandle:

• Customize ref value exposed to parent

• Use with forwardRef for imperative APIs

• Avoid overuse - prefer declarative patterns

useId:

• Generate unique IDs for accessibility

• SSR-safe (consistent client/server IDs)

• Perfect for form labels and ARIA attributes

useSyncExternalStore:

• Subscribe to external data sources safely

• Handles concurrent rendering correctly

• Use for browser APIs, third-party stores

⚠️ When to use:

• These are specialized hooks for specific use cases

• Most apps won't need them frequently

• Prefer standard hooks when possible
